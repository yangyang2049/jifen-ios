# 对阵板（纯队伍显示 / 可选胜负）

## 结论

场景**真实存在**，但**不要做成「拔河」专项**。同一类需求是：

> 大屏上清楚展示「谁对谁」；分数不是主信息，最多要「胜 / 负 / 进行中」。

现有缺口：没有「名字优先、分数可关」的现场对阵板。
最接近能力：简易计分、积分表、局域网展示端只读比分。

**产品名建议：对阵板**（中性，覆盖拔河 / 辩论 / 团建）。不要首发叫「拔河」。

## 权重

### 重（优先覆盖）

- 趣味运动会：拔河、二人三足、跳绳接力等——观众/裁判看清红白方；无分或一局定胜负
- 团建 / 班级 / 宿舍红白对抗——气氛板 + 身份认同
- 辩论赛、知识竞赛对阵——首屏是队名/正反方，不是分数
- 抽签对阵 / 下一场公示——候场大屏只要对阵

共同点：**队名、颜色、左右（红白）布局是核心**。

### 中

- 掰手腕、猜拳淘汰、一场定胜负类对抗
- 活动/电竞赛前「A vs B」待机页
- 局分记号 `○○●`（仍无 rally 分）——可作 v0 之后

### 轻（不必为它们开专项 / 已有替代）

- 完整球类计分 → 已有专项
- 联赛 W/D/L 账本 → 已有积分表
- 纯文字大屏 → 全屏弹幕
- 只读同步别人比分 → LAN display role

## 定位

```text
纯展示(队名+色) → 胜负标志 → 局分记号 → 简易计分(数字) → 完整运动规则
         ↑ 当前空档（拔河落在这里）
```

- **做**：一个通用「对阵板」模式
- **不做**：拔河 / 掰手腕 / 辩论各开一个目录项

## 建议 MVP（v0）

1. 双队大字队名，全屏计分板风格（红/蓝或主题色）
2. 仅展示，和/或点击标记胜负（细节开工时再定）
3. 目录放「其他」，挨着简易计分（默认倾向；可改）
4. 记录可只存队名 + 结果，不强行写数字比分

**非目标：** 多队网格、淘汰 bracket、复杂裁判状态、Watch 首发。
