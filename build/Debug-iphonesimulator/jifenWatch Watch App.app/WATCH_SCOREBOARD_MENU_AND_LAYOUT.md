# Watch 计分板菜单与布局参考（供鸿蒙等端对齐）

本文档从 Watch 计分板（`WatchSetBasedScoreboardView` / `WatchScoreboardView`）提炼菜单项、交互逻辑与布局方式，便于其他端（如鸿蒙）实现一致体验，并扩展竖屏/横屏布局切换。

---

## 一、菜单入口与手势

- **打开菜单**：上滑（Swipe up，`translation.height < -40`）。
- **撤销**：下滑（Swipe down，`translation.height > 40`）可直接撤销，无需打开菜单。
- 文案提示（`usage_prompt_message`）：  
  - 中文：`1. 点击对应半区加分\n2. 下滑撤销\n3. 上滑打开菜单`  
  - 英文：`1. Tap the corresponding half to add points\n2. Swipe down to undo\n3. Swipe up to open menu`

---

## 二、菜单项与逻辑

### 1. 撤销（Withdraw / Undo）

- **入口**：  
  - 菜单内「撤销」按钮；  
  - 比赛结束/暂停浮层上的「撤销」按钮（仅当 `undoButtonVisible && isFinished`）；  
  - 局间/局中休息浮层上的「撤销」按钮（当 `undoButtonVisible`）；  
  - 手势：下滑。
- **逻辑**：  
  - 每次可撤销的**状态变更前**保存快照（如每次 `addPoint` 前 `saveHistory()`）。  
  - 快照内容：红蓝分数、局数/盘数、是否暂停、是否结束、是否休息中、抢七、胜方、局中休息/决胜局换边等状态、是否已保存记录、以及**当前 actions 条数** `actionsCount`。  
  - 撤销时：从历史栈弹出最近一次快照，**整体恢复**到该快照（分数、局数、暂停/结束/休息等）；同时把 `actions` 截断到该快照的 `actionsCount`，保证时间轴与比分一致。  
  - 撤销后：播放撤销触觉（如 `WatchHaptics.shared.play(.undo)`）、将「撤销」按钮显示一段时间（如 5 秒）后自动隐藏。  
  - 若撤销的是「已保存记录」之后的状态，需将 `recordSaved` 置为 false，以便再次结束比赛时允许重新保存。

### 2. 结束比赛 / 暂停（End Game / Pause）

- **入口**：菜单内「暂停」/「继续」按钮。  
  - 未暂停时显示「暂停」；已暂停时显示「继续」。  
- **逻辑**：  
  - 点击「暂停」：  
    - `isStopped = true`，`isFinished = true`（视为本场结束）。  
    - 若尚未保存记录：根据当前局数判定胜方（`redSets > blueSets` → 红方胜等），保存比赛记录，播放结束触觉，显示「撤销」按钮（同撤销逻辑的显示时长）。  
  - 点击「继续」：  
    - `isStopped = false`，`isFinished = false`，可继续计分。  
  - 比赛结束/暂停后出现**浮层**：  
    - 标题：「比赛结束」或「比赛暂停」；若有胜方则显示一行结果文案（如「红方 2-1 获胜」，使用 `winner_result_format` 本地化）。  
    - 浮层上的「撤销」：同撤销逻辑，撤销后关闭撤销按钮并恢复可计分状态。  
    - 浮层上的「退出」：关闭计分页（dismiss），不重置分数。

### 3. 重置（Reset）

- **鸿蒙 Watch**：菜单内提供「重置」，调用 ViewModel `resetMatch()`：分数/局数/盘数归零，清除休息与结束状态，清空历史与 actions，不保存记录。  
- **iOS Watch**（已对齐）：菜单内提供「重置」，`resetMatch()` 行为与鸿蒙一致（清空分数/局/盘/休息/结束/历史/actions，重置计时，提示「已重置」）。

### 4. 切换布局（Switch Layout）

- **鸿蒙 Watch**：菜单内提供「布局」按钮，在竖屏（红上蓝下）与横屏（红左蓝右）间切换，偏好持久化。  
- **iOS Watch**（已对齐）：菜单内提供「竖屏」/「横屏」按钮，`WatchPreferences.scoreboardLayout` 持久化；body 根据 `scoreboardLayout` 使用 `VStack` 或 `HStack` 两半屏。

---

## 三、布局与 UI 要点（便于支持竖/横）

- **竖屏（Vertical）**：  
  - 上半区：红方（如 `Color(hex: 0xE53935)`），显示红方局数/盘数、当前局分数。  
  - 下半区：蓝方（如 `Color(hex: 0x1E88E5)`），显示蓝方局数/盘数、当前局分数。  
  - 每半区可点击加分，手势（上滑菜单、下滑撤销）在整屏生效。
- **横屏（Horizontal）**：  
  - 左半区：红方；右半区：蓝方。  
  - 显示与交互逻辑同竖屏，仅方向改为左右。
- 网球：除当前局分数外，还有盘数（games）和 set 小圆点等，两种布局下信息一致，仅排版方向不同。

---

## 四、其他与菜单相关的状态

- **局间休息 / 局中休息**：  
  - 休息浮层上可显示「撤销」按钮（与上述撤销逻辑一致），以及「继续比赛」/「关闭」结束休息。  
- **决胜局换边**：  
  - 羽毛球等有局中休息的项目，决胜局换边可用独立 overlay（如「决胜局换边」+「关闭」）；无局中休息的项目可用 toast/chip 提示（如「决胜局换边」）。  
- **抢七**：  
  - 网球抢七时，可在非菜单、非休息时显示「抢七」角标，不影响菜单与布局逻辑。

---

## 五、建议的鸿蒙端菜单项汇总

| 菜单项   | 说明 |
|----------|------|
| 撤销     | 与 Watch 一致：历史快照 + 恢复 + 截断 actions，撤销后短时显示撤销按钮。 |
| 结束比赛 | 与 Watch「暂停」一致：标记结束、保存记录、显示结束浮层（含撤销、退出）。 |
| 重置     | 清空本场分数与历史，回到 0-0，可选清空本场已保存标记。 |
| 切换布局 | 在竖屏（上红下蓝）与横屏（左红右蓝）间切换，计分板支持两种布局。 |

以上逻辑与 Watch 保持一致或在其基础上扩展，即可实现多端统一的菜单与布局体验。
